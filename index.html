Como funciona teoria

A janela de 5h

O Claude Code conta uma janela rotativa de 5 horas a partir da sua primeira mensagem do dia. Toda mensagem nesse intervalo consome a mesma cota. Quando a janela acaba, uma nova começa na próxima mensagem.

Se você começa às 09h sem warmup, sua janela 1 vai de 09h-14h e a janela 2 de 14h-19h. No expediente típico (9h-18h) você acessa 2 janelas, mas a segunda só usa 4h dos 5h disponíveis.

Por que warmup ajuda

Mandando uma única mensagem de ping automaticamente antes do seu expediente, você desloca o ciclo de resets pra dentro do horário de trabalho. Resultado: o reset acontece durante o expediente, te dando uma cota nova no meio do dia.

A regra simples é warmup = início_do_trabalho − 4h. Isso faz a primeira janela resetar 1h depois de você começar, a segunda janela cair 100% dentro do expediente, e a terceira começar a 4h do início do dia.

Tradeoffs honestos

  • O limite semanal continua valendo. Warmup não te dá mais cota total, só desloca quando os resets acontecem dentro do dia.
  • Cada warmup consome ~1 request do seu plano. Desprezível, mas existe.
  • Funciona melhor pra trabalho ≥ 6h. Pra jornadas curtas (≤5h), uma janela já cobre tudo.
  • Você precisa de uma máquina sempre ligada (server, mini PC, VPS) com o claude CLI instalado pra rodar o cron.

Pré-requisitos pro cron

  • Claude Code CLI instalado e autenticado na máquina onde o cron vai rodar
  • Path correto do binário — descubra com which claude e ajuste a linha gerada se diferente
  • Cron habilitado (Linux/macOS) ou Task Scheduler (Windows com adaptação manual)
